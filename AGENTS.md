# Codex 工作规则

## 项目目标

这个仓库用于承载可在电脑端和手机端 Codex 之间连续工作的项目内容。当前优先级是保持仓库清晰、可同步、可继续开发。

## 工作原则

- 改动要小而明确，避免一次性混入无关内容。
- 修改前先查看现有文件和 Git 状态。
- 不提交 `.env`、密钥、账号令牌或本地缓存文件。
- 如果新增技术栈，要同时更新 `README.md` 里的安装、运行和测试说明。
- 如果新增命令，要记录在本文件或 `README.md` 中，方便手机端 Codex 接手。

## 常用检查

当前还没有固定技术栈，所以没有统一的安装、启动或测试命令。后续添加项目代码时，请补齐：

- 安装依赖命令
- 本地启动命令
- 测试或检查命令
- 构建或发布命令

## 跨设备协作

- 从电脑切到手机前，先提交并推送改动。
- 从手机切回电脑前，先确保 GitHub 上的改动已经合并或提交。
- 遇到冲突时，优先保留最近确认过的业务内容，不要直接覆盖用户改动。
