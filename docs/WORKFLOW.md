# 工作流程

## 电脑端

1. 打开本地工作区。
2. 确认 GitHub 上没有更新需要同步。
3. 完成一段明确的改动。
4. 提交并推送到 GitHub。

## 手机端 Codex

1. 在 Codex 中选择 `fk911cat/hello-gitub` 仓库。
2. 让 Codex 基于最新分支继续工作。
3. 完成后提交改动，或创建可检查的 Pull Request。

## 设备切换规则

- 切换设备前，先把当前设备的改动提交到 GitHub。
- 不确定是否有未保存内容时，先查看 Git 状态。
- 如果出现冲突，先停下来确认哪些内容应该保留。

## 后续补齐

当项目开始使用具体技术栈后，在这里补充：

- 安装依赖
- 本地启动
- 测试命令
- 构建命令
- 发布或部署步骤
